# BOntoLink 后端模块化改造总结

## ✅ 完成的工作

### 1. 创建四层模块架构

```
backend/
├── bontolink-data/         ← 数据访问层（23 Mapper + 8 Entity + Flyway）
├── bontolink-base/         ← 基础设施层（数据源管理、连接池）
├── bontolink-admin/        ← 管理后台服务（端口 8088）
└── bontolink-ontology/     ← 本体服务（端口 8089，待实现）
```

### 2. 数据源管理提取到 bontolink-base

从 `bontolink-admin` 提取到 `bontolink-base/datasource/`：
- **DataSourceConnector**：数据源连接工具（支持连接池和直连）
- **DynamicDataSourceRegistry**：动态数据源连接池注册表（HikariCP）
- **DynamicPoolProperties**：连接池配置属性
- **PoolMetrics**：连接池运行时指标

### 3. 依赖关系调整

```
bontolink-data (数据访问)
    ↓ 依赖
bontolink-base (基础设施)
    ↓ 依赖
bontolink-admin / bontolink-ontology (应用层)
```

### 4. 包名规范

- 数据层：`com.beiktech.bontolink.data.*`
- 基础设施层：`com.beiktech.bontolink.base.*`
- 管理后台：`com.beiktech.bontolink.*`
- 本体服务：`com.beiktech.bontolink.ontology.*`

## 🎯 设计原则

### 1. 单一职责
- **data**：只负责数据访问，不包含业务逻辑
- **base**：提供与本体无关的通用基础设施
- **admin**：本体元数据管理
- **ontology**：本体推理和查询

### 2. 依赖方向
- 低层模块（data/base）不依赖高层模块（admin/ontology）
- 应用层模块通过 base 层共享公共功能

### 3. 边界清晰
- 数据源管理在 base 层，供所有应用模块使用
- Mapper/Entity 在 data 层，通过 Spring 自动配置注入
- 业务逻辑在应用层，不下沉到基础设施层

## 🔧 关键改进

### 移除 DictCacheHelper 依赖
**问题**：`DynamicDataSourceRegistry` 原依赖 `DictCacheHelper`（业务层服务）  
**解决**：移除字典校验逻辑，改为直接检查 `ds_type` 字段和 MongoDB 排除

**改动前**：
```java
if (dictHelper.exists("sys_ds_type", ds.getDsType())) {
    // 创建连接池
}
```

**改动后**：
```java
if (!"mongodb".equalsIgnoreCase(ds.getDsType())) {
    // 创建连接池
}
```

**理由**：
1. 数据源类型应在数据库层通过外键约束保证合法性
2. 基础设施层不应依赖业务层服务
3. 避免循环依赖和启动顺序问题

**实现方案**：
```
backend/
└── bontolink-starter/      ← 统一启动器

    └── src/main/

```

通过 Spring Boot Conditional 注解根据配置动态启用模块。

## 🚀 快速启动

### 编译所有模块
```bash
cd backend
mvn clean install
```

### 启动管理后台
```bash
cd backend/bontolink-admin
mvn spring-boot:run
# 访问 http://localhost:8088/bontolink/api/health
```

### 启动本体服务
```bash
cd backend/bontolink-ontology
mvn spring-boot:run
# 访问 http://localhost:8089/bontolink-ontology/actuator/health
```

## 📌 注意事项

1. **Mapper 扫描**：由 `bontolink-data` 的 `BontoLinkDataAutoConfiguration` 统一管理
2. **连接池配置**：在 `application.yml` 配置 `bontolink.datasource.pool.*`
3. **数据源初始化**：`DynamicDataSourceRegistry` 在 `@PostConstruct` 自动扫描 `sys_data_source` 表
4. **本地仓库**：模块 JAR 已安装到 `d:/maven/repository/com/beiktech/`
