# 分组树功能修复说明

## 问题描述

用户反馈：点击左侧分组树时，枚举类型和值类型列表没有根据分组进行过滤显示。

## 根本原因

1. **UI 缺失**：虽然后端和前端数据层都已支持分组功能，但页面模板中没有渲染 `LeftGroupTree` 组件
2. **数据未关联**：前端加载的枚举/值类型对象中没有 `group_id` 字段（需要从 `ont_biz_group_ref` 表关联）
3. **过滤逻辑未激活**：虽然代码中存在 `activeGroupId` 和分组过滤逻辑，但因为 UI 不存在，过滤逻辑从未被触发

## 解决方案

### 1. 枚举类型页面 (EnumTypes.vue)

**添加的功能**：
- ✅ 在行业分类树上方添加 `LeftGroupTree` 组件
- ✅ 实现 `onGroupChange` 回调，更新 `activeGroupId`
- ✅ 数据加载时从 `ont_biz_group_ref` 获取分组关联，将 `group_id` 注入枚举对象
- ✅ 在 `filteredTypes` 计算属性中添加分组过滤逻辑（递归收集子分组）
- ✅ 复用现有的 `openCreateGroup` 和 `submitGroup` 函数

**代码位置**：
- 模板：第 23-28 行（添加 LeftGroupTree）
- 导入：第 823 行（添加 LeftGroupTree 导入）
- 事件处理：第 1294-1297 行（onGroupChange 函数）
- 过滤逻辑：第 1114-1125 行（已存在，现在能被触发）

### 2. 值类型页面 (ValueTypes.vue)

**添加的功能**：
- ✅ 在行业分类树上方添加 `LeftGroupTree` 组件
- ✅ 实现 `onGroupChange` 和 `openCreateGroup` 回调
- ✅ 修改 `load()` 函数，加载所有分组并注入 `group_id`
- ✅ 在 `filtered` 计算属性中添加分组过滤逻辑
- ✅ 添加新建分组的模态对话框

**代码位置**：
- 模板：第 36-42 行（添加 LeftGroupTree）、第 372-401 行（新建分组模态）
- 导入：第 380 行（添加 LeftGroupTree 导入）
- 变量：第 455 行（activeGroupId）
- 事件处理：第 492-516 行（onGroupChange/openCreateGroup/submitGroup）
- 数据加载：第 590-610 行（修改 load 函数）
- 过滤逻辑：第 520-534 行（添加分组过滤）

### 3. 分组树组件特性 (LeftGroupTree.vue)

已存在的功能：
- 树形分组显示
- 搜索过滤
- 成员计数（显示每个分组下的枚举/值类型数量）
- 选中状态（高亮当前选中分组）
- 宽度拖拽（可调整左侧面板宽度）
- 新建分组按钮

## 测试方法

### 1. 验证枚举类型分组过滤

1. 启动后端：`cd backend && mvnw.cmd spring-boot:run`
2. 打开前端：访问枚举类型管理页面
3. 观察左侧：现在有两个树
   - 第一个：**分组树**（按业务组织划分）
   - 第二个：**行业分类树**（按行业领域划分）
4. 点击分组树中的任意分组
5. 验证：右侧列表只显示该分组及其子分组下的枚举
6. 点击"全部"：显示所有枚举

### 2. 验证值类型分组过滤

1. 打开值类型管理页面
2. 观察左侧：同样有分组树和行业分类树
3. 点击分组树中的任意分组
4. 验证：右侧列表只显示该分组及其子分组下的值类型
5. 测试分组计数：分组树中每个分组显示的数字应与实际成员数一致

### 3. 验证新建分组

1. 点击分组树顶部的"+"按钮
2. 填写分组名称、选择所属领域和父分组
3. 保存后验证：
   - 分组树中出现新分组
   - 可以选择该分组进行过滤
   - 在新建/编辑枚举或值类型时，可以选择该分组

### 4. 验证递归过滤

1. 创建父子分组结构：
   - 父分组：基础数据
   - 子分组：行政区划、组织机构
2. 给子分组添加枚举/值类型
3. 点击父分组"基础数据"
4. 验证：显示父分组和所有子分组的枚举/值类型

## 数据流程

```
用户点击分组
    ↓
onGroupChange(groupId)
    ↓
activeGroupId.value = groupId
    ↓
filteredTypes / filtered 计算属性重新计算
    ↓
递归收集该分组及所有子分组的 ID
    ↓
过滤列表：list.filter(item => groupIds.has(item.group_id))
    ↓
更新页面显示
```

## 与行业分类树的区别

| 特性 | 分组树 (LeftGroupTree) | 行业分类树 (CategoryTreeFilter) |
|------|----------------------|-------------------------------|
| 数据来源 | `ont_biz_group` + `ont_biz_group_ref` | `ont_industry_category` |
| 过滤字段 | `group_id` | `category_code` |
| 组织方式 | 业务功能分组（自定义） | 行业领域分类（标准） |
| 适用场景 | 项目内部资源组织 | 跨项目行业标准分类 |
| 支持新建 | ✅ 是 | ❌ 否（标准数据） |

## 注意事项

1. **幂等性**：多个过滤条件同时生效（分组 + 行业分类 + 搜索）
2. **性能**：分组过滤使用 Set 查找，时间复杂度 O(n)
3. **兼容性**：保留了旧代码中的 `activeGroupId` 和 `selectGroup` 函数
4. **未来增强**：
   - 分组拖拽排序
   - 批量移动枚举/值类型到其他分组
   - 分组权限控制
