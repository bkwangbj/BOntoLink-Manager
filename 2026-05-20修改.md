## 3. 自定义图标编组 + SVG 批量上传

### 3.1 持久化层 [`bl.js`](frontend/src/lib/bl.js)
- localStorage Key:`bl.iconlib.custom.v1`
- 结构:`{ groups: [{ key, label, iconIds }], icons: { [id]: { name, viewBox, content } } }`
- 导出:`getCustomIconData / setCustomIconData / onCustomIconsChange`
- `custom:<id>` 命名空间,被 `BL.icon()` 自动识别

### 3.2 选择器侧栏卡片化
- **我的图库**(自定义,置顶)+ **内置图标库** 两大卡片
- 卡片化样式:白底 + 边框 + 圆角 + 标题栏浅灰底 + 底分隔线
- 标题栏可点击折叠/展开,chevron 旋转 0°/90°
- 计数 badge 描边样式
- 侧栏宽度 160px → **200px**,所有文字 `white-space: nowrap` + 省略号

### 3.3 自定义编组管理
- 新建编组:`+` 按钮 → `BL.prompt` 输入名称
- 行内重命名:点击 ✎ 进入 input 编辑(Enter 提交、Esc 取消、Blur 提交)
- 删除编组:trash 按钮 → `BL.confirm` 二次确认,连带删除编组下所有图标
- hover 仅切换 `display`,通过强制 20×20 + `min-height` 锁定行高,不抖动

### 3.4 SVG 批量上传
- 选中自定义编组后,工具栏出现「上传 SVG」按钮
- 多选 `.svg`,逐个解析:
  - 抽出 `viewBox`(或从 `width/height` 推算)
  - 序列化所有子节点为字符串
  - `fill="#xxx"` / `stroke="#xxx"` 替换为 `currentColor`
  - 内联 `fill: #xxx` / `stroke: #xxx` 同样替换
- 文件名(去 `.svg`)作为图标名
- 失败计数提示

### 3.5 批量操作
- 工具栏「批量操作」按钮(仅自定义编组可见)
- 进入后:已选 N/M · 全选 · 取消选择 · **删除(N)** · 退出批量
- 单击切换选中状态;选中格高亮蓝边 + 浅蓝底 + 右上勾标
- 批量删除前 `BL.confirm` 二次确认

## 5. 自定义 SVG 颜色染色修复

### 问题
上传的 SVG 应用到节点后,颜色不跟随节点配色 — 显示成 `currentColor` 解析到了文本色。

### 修复
[`bl.js`](frontend/src/lib/bl.js) 渲染时同时设置:
- `fill="${color}"` — 子元素无 fill 时继承
- `style="color:${color}"` — 让 `currentColor` 正确解析

两条路径都指向用户传入色值,确保染色生效。
