## 二、命名空间的作用

### 1. **语义唯一性**
防止不同领域的同名概念冲突:
```
http://ontology.example.com/water-service#Project  (水利项目)
http://ontology.example.com/construction#Project   (建设项目)
```

### 2. **RDF/OWL 标准兼容**
生成标准的 OWL 本体时需要完整的命名空间 URI:
```xml
<owl:Class rdf:about="http://ontology.example.com/water-service#WaterServiceCompany">
  <rdfs:label>水利服务企业</rdfs:label>
</owl:Class>
```

### 3. **跨系统引用**
其他系统可以通过 URI 引用本系统的概念:
```sparql
PREFIX ws: <http://ontology.example.com/water-service#>

SELECT ?company ?capital
WHERE {
  ?company a ws:WaterServiceCompany ;
           ws:capital ?capital .
  FILTER(?capital > 1000000)
}
```

## 三、BOntLink-Manager 的命名空间设计

### 数据表结构

**ont_biz_namespace** (命名空间定义)
```sql
CREATE TABLE ont_biz_namespace (
  ns_code       VARCHAR(64),   -- 命名空间代码 (如 "water_service")
  ns_name       VARCHAR(128),  -- 命名空间名称 (如 "水利服务业本体")
  ns_uri        VARCHAR(255),  -- 命名空间URI (如 "http://...")
  rdfs_label    VARCHAR(255),  -- RDFS标签
  rdfs_comment  VARCHAR(255),  -- RDFS注释
  ...
);
```

**ont_class** (对象类型)
```sql
CREATE TABLE ont_class (
  id           VARCHAR(64),
  api_name     VARCHAR(128),  -- OWL 类名 (如 "WaterServiceCompany")
  ns_code      VARCHAR(64),   -- 关联命名空间
  ...
);
```

### 关系映射

```
ont_biz_namespace.ns_code ←→ ont_class.ns_code
```

## 四、转换脚本的处理逻辑

### convert-to-bonotlink-ui-full.ps1

```powershell
# 1. 加载命名空间
GET /api/namespaces
→ $nsMap[ns_code] = namespace

# 2. 加载对象类型
GET /api/class-meta/classes
→ 每个类包含 ns_code

# 3. 转换实体时关联命名空间
foreach ($cls in $allClasses) {
    $entity = @{
        name = $cls.display_name
        owlClass = $cls.api_name
        namespace = $nsMap[$cls.ns_code]  # 关联命名空间
    }
}

# 4. 输出顶层命名空间定义
$ontology = @{
    namespaces = $nsMap.Values        # 顶层命名空间列表
    entities = $entities               # 每个实体包含 namespace 字段
}
```

## 六、使用场景

### 场景1: 导出标准OWL本体

```java
// 使用命名空间信息生成标准 OWL
public String generateOWL(OntologyDefinition ontology) {
    StringBuilder owl = new StringBuilder();
    owl.append("@prefix : <").append(namespace.getUri()).append("> .\n");
    
    for (Entity entity : ontology.getEntities()) {
        owl.append(":").append(entity.getOwlClass()).append(" a owl:Class ;\n");
        owl.append("  rdfs:label \"").append(entity.getName()).append("\" .\n");
    }
    
    return owl.toString();
}
```

输出:
```turtle
@prefix : <http://ontology.example.com/water-service#> .

:WaterServiceCompany a owl:Class ;
  rdfs:label "水利服务企业" .
```

### 场景2: 跨本体引用

```json
// 引用其他命名空间的概念
{
  "entities": [
    {
      "name": "水利项目",
      "namespace": {
        "code": "water_service",
        "uri": "http://ontology.example.com/water-service#"
      },
      "fields": [
        {
          "name": "承包企业",
          "type": "REFERENCE",
          "referenceTo": {
            "namespace": "water_service",
            "class": "WaterServiceCompany"
          }
        }
      ]
    }
  ]
}
```

### 场景3: SPARQL 查询

```sparql
PREFIX ws: <http://ontology.example.com/water-service#>
PREFIX geo: <http://www.w3.org/2003/01/geo/wgs84_pos#>

SELECT ?company ?location
WHERE {
  ?company a ws:WaterServiceCompany ;
           ws:companyName ?name ;
           geo:location ?location .
}
```
