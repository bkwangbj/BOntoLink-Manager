# 枚举到值类型同步工具

## 功能说明

为所有枚举类型自动创建对应的值类型（如果尚不存在）。

## 使用方法

### 方式一：使用 HTML 工具页面

1. **启动后端服务**
   ```powershell
   cd f:\aiProject\BOnotLink-Manager\backend
   mvnw.cmd -q -DskipTests spring-boot:run
   ```

2. **打开同步工具页面**
   - 在浏览器中打开：`f:\aiProject\BOnotLink-Manager\frontend\sync-enum-to-valuetype.html`
   - 或者直接双击该文件

3. **点击"开始同步"按钮**
   - 工具会自动调用后端 API 进行同步
   - 完成后显示同步结果（新建数量、跳过数量等）

### 方式二：直接调用 API

**端点**: `POST http://localhost:8088/bontolink/api/value-types/sync-from-enums`

**请求示例**（使用 curl）:
```bash
curl -X POST http://localhost:8088/bontolink/api/value-types/sync-from-enums
```

**响应示例**:
```json
{
  "code": 200,
  "msg": "ok",
  "data": {
    "total_enums": 15,
    "created_count": 10,
    "skipped_count": 5,
    "created_api_names": [
      "industry_code",
      "project_type",
      "water_quality_level",
      ...
    ]
  }
}
```

### 方式三：在浏览器控制台调用

1. 打开项目前端页面（任意页面）
2. 按 F12 打开开发者工具
3. 在 Console 中执行：
   ```javascript
   fetch('http://localhost:8088/bontolink/api/value-types/sync-from-enums', {
     method: 'POST',
     headers: { 'Content-Type': 'application/json' }
   })
   .then(r => r.json())
   .then(data => console.log('同步结果:', data))
   ```

## 同步逻辑

1. **扫描所有枚举类型**：从 `ont_enum_types` 表读取所有枚举
2. **检查现有值类型**：查找 `constraint_type = 'Enum'` 的值类型，提取已关联的 `enum_id`
3. **创建缺失的值类型**：
   - 为每个没有对应值类型的枚举创建值类型
   - 继承枚举的 `api_name`、`rdfs_label`、`category_code`
   - `base_type` 设为 `'String'`
   - `constraint_type` 设为 `'Enum'`
   - `enum_id` 关联到枚举 ID
4. **跳过已存在的**：如果枚举已有对应值类型，不重复创建
5. **返回统计信息**：新建数量、跳过数量、创建的 API 名称列表

## 注意事项

- ✅ **幂等操作**：多次执行不会重复创建，已存在的会被跳过
- ✅ **不影响已有数据**：只创建缺失的值类型，不修改现有值类型
- ⚠️ **API 名称冲突**：如果枚举的 `api_name` 与现有值类型冲突，该枚举会跳过（不会覆盖）
- ⚠️ **需要后端运行**：确保后端服务已启动在 `localhost:8088`
